## 3.20.0 (2026-03-29)

### Changes
- Added `regimes/` top-level directory with 10 governance regimes
- Added Founding Principle #23: Regime-Based Governance
- Added Governance Layer to architecture (between Constitutional and Execution)
- Added `hooks/handlers/regime-runner.ts` — core enforcement engine
- Added `tools/regime-check.ts` — CLI compliance audit tool
- Integrated regime checks into session-end hook (with warning cache for pre-prompt)
- 10 regimes: documentation, secrets, skill-hygiene, skill-index-integrity, capabilities-manifest, data-source-registry, project-metadata, doc-integrity, memory-ownership, hook-latency
